🌐 ¿Qué son las Leyes UX/UI?

Las Leyes UX/UI son reglas, principios y fundamentos que se usan para diseñar páginas web y aplicaciones fáciles de usar, atractivas y organizadas.

Estas leyes ayudan a entender:

Cómo piensa una persona al usar una web
Cómo interactúa con botones y menús
Qué diseño resulta cómodo
Cómo evitar confusión
Cómo mejorar la experiencia del usuario

Se utilizan en:

Diseño web
Aplicaciones móviles
Sistemas empresariales
Tiendas online
Videojuegos
Redes sociales
🧠 Primero: ¿Qué significa UX y UI?
🎯 UX = User Experience

(Significa: Experiencia del Usuario)

UX se enfoca en:

Qué siente el usuario
Si la página es fácil
Si encuentra rápido lo que busca
Si navegar es cómodo
Ejemplo:

Cuando entras a una web y:
✅ encuentras todo rápido
✅ entiendes cómo funciona
✅ no te pierdes

Eso es un buen UX.

🎨 UI = User Interface

(Significa: Interfaz de Usuario)

UI es la parte visual.

Incluye:

Colores
Botones
Tipografía
Iconos
Diseño
Espacios
Animaciones
Ejemplo:

Instagram tiene:

colores simples
botones ordenados
diseño limpio

Eso es UI.

📌 Diferencia entre UX y UI
UX	UI
Experiencia	Apariencia
Cómo funciona	Cómo se ve
Organización	Diseño visual
Facilidad de uso	Colores y estilo
🏛 ¿Qué son exactamente las leyes UX/UI?

Son principios psicológicos y visuales usados para:

mejorar páginas web
facilitar navegación
aumentar comodidad
hacer diseños modernos

Muchas vienen de:

psicología humana
comportamiento visual
diseño gráfico
interacción humano-computadora
🧩 Leyes UX/UI más importantes
1. 📌 Ley de Hick
¿Qué dice?

Mientras más opciones tenga una persona, más difícil será decidir.

Ejemplo:

Si una web tiene:

50 botones
demasiados menús
mucha información

El usuario se confunde.

Buen diseño:

✅ pocas opciones
✅ menú simple
✅ información clara

2. 🎯 Ley de Fitts
¿Qué dice?

Los botones grandes y cercanos son más fáciles de usar.

Ejemplo:

En celulares:

WhatsApp usa botones grandes
TikTok pone botones visibles
Importancia:

Ayuda a:

evitar errores
mejorar velocidad
facilitar navegación
3. 🧠 Ley de Jakob
¿Qué dice?

Los usuarios prefieren páginas similares a las que ya conocen.

Ejemplo:

La mayoría de webs tienen:

logo arriba izquierda
menú arriba
carrito arriba derecha

Porque las personas ya están acostumbradas.

4. 📍 Ley de Proximidad
¿Qué dice?

Los objetos cercanos se perciben relacionados.

Ejemplo:

En formularios:

nombre y apellido juntos
contraseña debajo del usuario

Esto mejora el orden.

5. ✨ Ley de Simplicidad

También llamada:

Ley de Prägnanz
¿Qué dice?

Las personas entienden mejor diseños simples.

Buen diseño:

✅ limpio
✅ organizado
✅ pocos colores

Mal diseño:

❌ mucho texto
❌ colores excesivos
❌ demasiadas imágenes

6. 👀 Efecto Estético-Usabilidad
¿Qué dice?

Las personas creen que algo bonito funciona mejor.

Aunque tenga pequeños errores, si se ve profesional:

genera confianza
parece más fácil
7. 🔁 Ley de Consistencia
¿Qué dice?

Todo debe mantener el mismo estilo.

Ejemplo:

Si un botón es azul:

todos los botones principales deberían ser iguales

Esto evita confusión.

8. ⚡ Ley de Tiempo de Respuesta

Las personas quieren rapidez.

Si una página tarda mucho:

abandonan la web
se frustran

Por eso:

las imágenes se optimizan
las páginas se hacen rápidas
9. 📚 Ley de Miller
¿Qué dice?

Las personas recuerdan poca información al mismo tiempo.

Normalmente:

entre 5 y 9 elementos

Por eso:
✅ menús pequeños
✅ información resumida

10. 🔍 Ley de Jerarquía Visual

Lo importante debe verse primero.

Ejemplo:
títulos grandes
botones resaltados
colores importantes

Ayuda a dirigir la atención.

🎨 Elementos básicos del UI
🎨 Colores

Los colores transmiten emociones.

Color	Significado
Azul	confianza
Verde	naturaleza
Rojo	peligro o atención
Negro	elegancia
Amarillo	energía
🔤 Tipografía

Es el estilo de letra.

Debe ser:
✅ clara
✅ legible
✅ ordenada

📏 Espaciado

El espacio ayuda a:

organizar
respirar visualmente
evitar saturación
🖼 Iconos

Ayudan a entender rápido.

Ejemplo:

🛒 carrito
🔍 búsqueda
❤️ favoritos
📱 Diseño Responsive

Una web debe verse bien en:

celular
tablet
computadora

Eso se llama:

Responsive Design
🧑‍💻 ¿Qué hace un diseñador UX/UI?

Se encarga de:

investigar usuarios
crear diseños
hacer pruebas
mejorar experiencia
diseñar interfaces
🔄 Proceso UX/UI
1. Investigación

Se estudia al usuario.

2. Wireframe

Boceto simple de la página.

3. Diseño UI

Se agregan colores y estilos.

4. Prototipo

Simulación interactiva.

5. Pruebas

Usuarios prueban el diseño.

6. Mejora

Se corrigen errores.

🧱 ¿Qué es un Wireframe?

Es un dibujo básico de la página.

Muestra:

dónde van botones
menús
imágenes
textos

Sin colores todavía.

🖥 ¿Qué es un Prototipo?

Es una versión interactiva.

Puedes:

tocar botones
navegar
simular la app
